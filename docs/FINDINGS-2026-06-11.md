## P0 — do naprawy w pierwszej kolejności

### P0-1. Parsowanie JSON z MiniMax jest kruche — POTWIERDZONE PRODUKCYJNIE
`services/minimax.py::generate_json()` robi wyłącznie: `json.loads(text)` + regex na markdown fences.
`pipelines/report.py::_normalize_item()` ratuje tylko przypadek „string zamiast dict w tablicy".

Tymczasem 2026-06-11, na żywym MiniMax M2.7 w siostrzanym serwisie (BEAUTY_AUDIT/convex/strategy),
złapaliśmy DWA powtarzalne tryby psucia odpowiedzi mimo `stop_reason=end_turn`:
1. **surowe znaki nowej linii wewnątrz wartości stringów** (nielegalne w JSON),
2. **polskie cytaty zamykane prostym cudzysłowem** — `„Ojciec + syn (włosy)"` — prosty `"` ucina
   string w połowie wartości.
Dodatkowo zaobserwowano wtrącenia obcojęzyczne (chiński token w polskim tekście).

Skutek w bagent: ValueError/JSONDecodeError → fail kroku → (w #1) fail pipeline'u albo silent
degradacja scoringu (free_report ma fallback `{score:10, issues:[]}`).

**Fix (port z BEAUTY_AUDIT, gotowy algorytm w convex/strategy/parse.ts):** przed json.loads
pipeline napraw: (a) wytnij bloki <think> i prozę przed/po JSON (slice do ostatniego `}`),
(b) zamień znaki kontrolne U+0000-U+001F na spacje, (c) escapuj nieescape'owane `"` w stringach
(heurystyka: `"` jest terminatorem tylko gdy następny niebiały znak ∈ {`,`,`}`,`]`,`:`}),
(d) domknięcie STOSOWE nawiasów + iteracyjne przycinanie ogona dla odpowiedzi uciętych.
Do tego reguła w promptach: zakaz prostych cudzysłowów w treści (pisać «...»), wyłącznie polski.

### P0-2. Silent fallback w restrukturyzacji kategorii
`pipelines/category_restructure.py`: wyjątek agent loopa → log warning → zwrot pustego mappingu.
Frontend nie odróżnia „agent uznał, że nic nie zmieniamy" od „agent się wywalił".
**Fix:** jawny status w wyniku (np. `restructure_status: ok|failed`) propagowany do raportu.

### P0-3. Webhooki Convex: best-effort bez retry + bez idempotencji
`services/convex.py::_post_webhook(raise_on_error=False)`; job oznaczany completed PRZED webhookiem;
brak backoffu i brak klucza idempotencji (arq max_tries=3 → przy retry taska webhook complete może
polecieć wielokrotnie). Skutek: klient z wiecznym spinnerem albo podwójne sygnały complete.
**Fix:** backoff (2/5/10 s) + nagłówek Idempotency-Key=job_id; rozważyć trwałą kolejkę webhooków.

## P1 — ryzyka i degradacje maskowane logami

- **P1-1. JobStore in-memory** (`job_store.py`): restart FastAPI = pusty dashboard i utrata metadanych
  jobów (arq w Redisie je ma, ale UI nie). Idempotencja `/api/competitor/report/refresh` oparta o ten
  lokalny store → **race** między check a enqueue (fix: Redis SET NX).
- **P1-2. Synteza konkurencji — fallback łańcuchowy degraduje po cichu**: deterministyczny fallback
  daje max 3 rekomendacje bez SWOT; raport wygląda na „kompletny", użytkownik nie wie, że dostał
  wersję awaryjną. **Fix:** flaga `synthesis_source: minimax|openai|deterministic` w report_data.
- **P1-3. `estimatedRevenueImpactGrosze` bez metodologii**: prompt mówi tylko „szacunkowy roczny wpływ
  (opcjonalne)"; longStrategy dolicza STAŁE (Q2=1.0M, Q3=1.5M, Q4=2.2M groszy). Liczby mogą być
  halucynacją; brak walidacji zakresów. **Fix:** zdefiniować wzór w prompcie (np. delta ceny ×
  szacowany wolumen z reviews) + sanity-check zakresu + disclaimer we froncie.
- **P1-4. Selekcja konkurencji — ciche pomijanie kandydatów**: brak focus bundle subjecta → degradacja
  do scoringu v1 (tylko warning); brak bundle kandydata → kandydat pomijany (setki mogą wypaść bez
  śladu w raporcie). **Fix:** licznik pominiętych w meta raportu.
- **P1-5. SWOT sanitizer dropuje punkty po cichu** (`competitor_synthesis.py`): punkty z błędnymi
  sourceDataPoints znikają; możliwy SWOT z 1 punktem bez ostrzeżenia. **Fix:** licznik dropów + retry
  pojedynczego punktu.
- **P1-6. Timeout 120 s MiniMax vs interleaved thinking**: duże cenniki (180+ usług) → thinking potrafi
  zjeść 40-60 s; retry podwaja koszt czasowy (~4 min ciszy dla frontendu). **Fix:** progress callback
  przed/po każdym wywołaniu + rozważyć podniesienie timeoutu dla kroków agentowych.
- **P1-7. Brak fallbacku LLM w BAGENT #1**: synteza konkurencji ma OpenAI fallback, raport audytu nie.
  Awaria MiniMax = fail całego audytu klienta. **Fix:** ten sam wzorzec gpt-4o-mini structured outputs.
- **P1-8. Taxonomy Pass 5**: default `openai` (rozjazd z narracją „MiniMax-first" w CLAUDE.md);
  chunk=30 to magic number po incydencie 192→10 decyzji; brak dynamicznego dobierania chunku.
- **P1-9. Cancel = polling**: `bagent:cancel:{job_id}` sprawdzany tylko na checkpointach; krok z długim
  wywołaniem LLM nie przerwie się minutami.
- **P1-10. Klienci bez poolingu**: SupabaseService/MiniMaxClient tworzone per task zamiast w
  `startup(ctx)` workera; literówka w env wybucha w środku joba zamiast na starcie.
- **P1-11. free_report.py bez wymuszenia zamrożenia**: kontrakt „nie zmieniać" trzyma się tylko
  dyscypliny; brak testu CI porównującego ze snapshotem.
